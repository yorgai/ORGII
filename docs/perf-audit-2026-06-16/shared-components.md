# Performance Audit — Shared / Design-System Components

`docs/perf-audit-2026-06-16/shared-components.md` · branch `ORGII-dev` · read-only, no source changed

## Scope covered

- **103 component folders** under `src/components/` enumerated; grepped all of them for the key signatures (`.Provider value=`, `createContext`, `React.memo`, `.sort/.filter/.reduce`, inline `value={…}` / `style={{`).
- **Read in full** (to confirm, not flag-from-grep): `Button`, `Tag`, `Tooltip`, `Message`, `Menu`, `Table/TableBody`, `StackListRow`, `IconButton`, `ChatBubble`, `Avatar`, `Select/useSelect`, `Dropdown/DropdownItem`, plus the 6 context providers (`Menu`, `Collapse`, `Form`, `Checkbox`, `Radio`, `FileTreeContent`).

## Summary

Nothing here is a catastrophic hot-loop. The most "textbook" issues are **un-memoized React Context `value` objects** in the group/compound components (`Menu`, `Checkbox.Group`, `Radio.Group`, `Collapse`, `Form`) — every provider re-render hands consumers a fresh object and re-renders the whole subtree. `Button`'s per-render closure/style pattern is **real but low per-instance cost**; it ranks high only because `Button` renders so often. Several primitives are already done correctly (`IconButton`, `Avatar`, `DropdownItem` are `memo`'d; `useSelect` uses `useMemo`; `FileTreeContent` memoizes its provider value) — good references for the fixes below.

**Is `Button` the worst offender?** No — it's _comparable_, not uniquely bad. The context-provider omissions have larger theoretical fan-out; `Button`'s edge is sheer render frequency. None is dramatically worse than the others.

## High — real re-render fan-out, worth fixing

| Component (file:line)                           | Pattern                                                                                                | Why it costs                                                                                                                         | Suggested fix                                                                                                                                            |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Menu` (`Menu/index.tsx:328-337`)               | `MenuContext.Provider value={contextValue}` where `contextValue` is a fresh object literal each render | Every `Menu` render (incl. parent-driven re-renders) gives all `MenuItem`/`SubMenu` consumers a new context identity → all re-render | `useMemo(() => ({selectedKeys, openKeys, mode, onSelect, onOpenChange}), [...])`. Handlers are already `useCallback` so the memo will actually be stable |
| `Checkbox.Group` (`Checkbox/index.tsx:413-428`) | `CheckboxGroupContextValue` object literal passed to Provider                                          | All checkboxes in the group re-render on any group re-render; payoff scales with group size                                          | `useMemo` the value; `onChange` is already `useCallback`'d                                                                                               |
| `Radio.Group` (`Radio/index.tsx:301-320`)       | Same — fresh `contextValue` object each render                                                         | Same fan-out across all radios                                                                                                       | `useMemo` the value (deps: `value, disabled, type, size, handleRadioChange`)                                                                             |
| `Collapse` (`Collapse/index.tsx:296-324`)       | Same — fresh `contextValue` each render                                                                | All `Collapse.Item` consumers re-render                                                                                              | `useMemo`; `handleItemChange` already `useCallback`'d                                                                                                    |

These four are the same one-line fix and the same root cause; `FileTreeContent/index.tsx:170-177` already demonstrates the correct `useMemo` pattern. **Honesty note:** in practice the fan-out is **bounded** (when group state changes, consumers _should_ re-render anyway). The win is avoiding re-renders triggered by _unrelated_ parent renders / new `children`. Worth doing because it's nearly free, but don't expect dramatic numbers unless groups are large.

## Medium — real but lower-frequency / lower per-instance cost

| Component (file:line)                                            | Pattern                                                                                                                                                      | Why it costs                                                                                                                                         | Suggested fix                                                                                                                                                                                          |
| ---------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `Button` (`Button/index.tsx:294-316`, called `362`)              | `getBorderRadius`/`getIconOnlySize`/`getButtonStyles` re-created every render; `getButtonStyles()` returns a brand-new `React.CSSProperties` each render     | Per-instance cost is tiny; adds up because `Button` is everywhere incl. list rows. New `style` object also means a DOM `style` reconcile each render | Hoist the shape→radius map to module scope; compute the style object with `useMemo` keyed on `size, shape, iconOnly, long, children-presence, icon, loading, style`. Optional: wrap `Button` in `memo` |
| `Tag` (`Tag/index.tsx:141`, style `231-238`, handlers `255/271`) | Pure presentational, rendered in lists, **not** `React.memo`; rebuilds `tagClasses`/`tagStyle` and inline `createKeyboardActivationHandler(...)` each render | Re-renders with every parent render even when props are unchanged; common in filter/label lists                                                      | Wrap export in `React.memo`; the class/style rebuild is cheap enough to leave                                                                                                                          |
| `StackListRow` (`StackListRow/index.tsx:33`)                     | Presentational list row, **not** memoized                                                                                                                    | Re-renders on every parent render in stack/tree lists                                                                                                | `React.memo` (props are mostly primitives + nodes)                                                                                                                                                     |
| `Form` (`Form/index.tsx:225-232`)                                | Provider `value` object literal not memoized                                                                                                                 | `FormItem`s re-render on any `Form` re-render                                                                                                        | `useMemo`. Lower than the High group because the value (`layout/labelAlign/colon`) is effectively static and Form rarely re-renders                                                                    |
