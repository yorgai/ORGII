## Notes on Symptom vs Debt

The user pasted the dom-element JSON for the OUTER container (line 355's `<div>`). The visible debt at that container is exactly **one** D4 issue: clickable-div without keyboard. The user's instinct ("audit this UI") is one-third right — there IS a real fix in the pasted element, and it's a11y, not styling.

But the densest debt in the file is two scroll-levels DOWN inside `CachedFileChip` (lines 106-165) — the cached-file preview popover. That code has 3 D1 + 3 D2 + 3 D3 + 2 D4 hits in ~60 LOC. The user can't see it from the pasted element because the popover only renders on chip click, and the pasted screenshot was an unexpanded message. Flag the popover to the user even though it isn't in the screenshot — it's where the real cleanup ROI is.

Distinct from the prior memory `workspace_user_chat_item_truncation.md`: that one documents the 72/240px expand/collapse behavior at lines 410-413 (which this audit verdicts as `keep` with `D3` justification). No regression there.
